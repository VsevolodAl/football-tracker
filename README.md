# ⚽ Футбольный трекер 2026/27

Трекер матчей и таблиц **Арсенала**, **Спартака** и **Факела**.
Данные загружаются на лету через serverless-функцию на Vercel — при каждом
открытии страницы (с кэшем на стороне Vercel на 30 минут) или по нажатию
кнопки «Обновить».

---

## Архитектура

```
index.html      # Сам сайт — фронтенд, тянет данные с /api/data
api/data.js     # Vercel serverless-функция:
                #   - Арсенал: football-data.org (нужен FOOTBALLDATA_KEY)
                #   - Спартак / Факел: живой скрейпинг календаря матчей
                #     с sports.ru (api-football не даёт доступа к сезону
                #     2026/27 на бесплатном плане)
vercel.json     # Настройки кэширования ответа /api/data
package.json    # Зависимости serverless-функции (cheerio)
```

## Деплой на Vercel

1. Зайдите на [vercel.com](https://vercel.com) → **Add New → Project** → выберите этот репозиторий.
2. В **Settings → Environment Variables** добавьте:

   | Имя | Значение |
   |-----|----------|
   | `FOOTBALLDATA_KEY` | Ваш токен с [football-data.org](https://www.football-data.org/client/register) |

   (Без него данные Арсенала не подтянутся — Спартак и Факел от него не зависят.)
3. Deploy. Сайт будет доступен на `https://ВАШ-ПРОЕКТ.vercel.app/`.

## Локальный запуск

```bash
npm install
vercel dev
```

## Обновление данных

Данные не хранятся в репозитории — `/api/data` каждый раз собирает их заново
(с кэшем Vercel на 30 минут). Кнопка «Обновить» в шапке сайта запрашивает
`/api/data` ещё раз.
